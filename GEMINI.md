# GEMINI.md

このファイルは Gemini CLI がこのリポジトリで作業する際のガイダンスを提供します。

## ユーザーの興味領域

- arixiv等に掲載されるLLM・基盤モデルの論文・研究動向
- NVIDIA / Google / Amazon / Meta / Apple および主要AI企業の動向
- AIエージェント（LangChain / LangGraph / AutoGen / CrewAI等）

## 収集・要約ルール

| 項目 | 仕様 |
|------|------|
| 収集期間 | 過去7日以内に公開された記事のみ |
| 収集件数 | 最大5件/回 |
| 日本語要約 | 600字以内 |
| 出力形式 | 1行1記事のJSONL をプロンプトで指定された**ファイルに書き出す** |

## JSONL フィールド定義

```json
{"title": "記事タイトル", "link": "URL", "description": "概要", "date": "公開日", "source": "ソース名", "category": "カテゴリ", "fetched_at": "収集日時ISO8601", "summary_ja": "600字以内の日本語要約", "url_status": 200}
```

## カテゴリ定義

`category` フィールドには**以下の5つのいずれか1つのみ**を使用すること。それ以外の値は使用禁止。

| カテゴリ | 対象 |
|----------|------|
| `LLM` | モデルリリース・ベンチマーク・LLM研究・論文 |
| `Agents` | Agentフレームワーク・マルチエージェント・LangChain/CrewAI/AutoGen等 |
| `Business` | 企業動向・資金調達・M&A・製品リリース・市場分析 |
| `Infrastructure` | GPU/チップ・クラウドインフラ・MLOps・ハードウェア |
| `Safety` | AI Safety・Alignment・規制・訴訟・倫理 |

`url_status` には `google_web_search` で確認したHTTPステータスコード（整数）を記録すること。**200番台のもののみ** `link` および `url_status` フィールドに値を設定してJSONLに出力すること。それ以外のステータスコードの記事は出力しないこと。

## 実行指示

ワークフローから渡されるプロンプトに従い、以下を実行せよ：

1. 興味領域に関連する最新のAI/LLMニュース・論文を調査する
2. 収集済みURLに含まれる記事はスキップする
3. 各記事について、**カテゴリ定義の表を参照し、最も適切な1つを `category` フィールドに設定する**（表にない値は使用禁止）
4. 各記事を600字以内で日本語要約し、プロンプトで指定された**出力先ファイルにJSONL形式で書き出す**（1行1記事）

## URL に関する厳守事項

- `link` フィールドには**検索結果で実際に確認したURLのみ**を使用すること
- URLを推測・補完・生成してはならない（ハルシネーション禁止）
- `vertexaisearch.cloud.google.com` などのリダイレクトURLは使用禁止。リダイレクト先の**最終的な記事URLを使用**すること

## 品質チェック

- 記事をJSONLに書き出す前に、`google_web_search` でそのURLを直接検索し、**検索結果にそのURLが実際に表示されること**を確認すること
- URLは検索結果から得たものをそのままコピーすること。推測・補完・生成は禁止
